# fkf 정회원 포털 — 데이터(전용 명단 탭) 셋업 가이드

fkf 포털은 마스터 시트 **`fKF_회원 관리`**(`112_MplXH5gCAKODQyDcvymu8GsWav180zqcmq187jCQ`)에서
회원 명단을 읽습니다. 이 시트의 라이브 원본 탭 **`MDB`**(회원 데이터가 21행부터
시작하고 실적이 섞여 있음)는 앱이 바로 읽기 어렵기 때문에, **헤더가 1행에
정리된 전용 명단 탭 `포털명단`을 새로 만들어** 그 탭을 읽도록 합니다.

> 조회용 `fKF 정회원 이력조회` 시트가 `MDB`를 IMPORTRANGE로 참조하는 것과 같은
> 원본을, 포털이 읽기 좋은 평평한 표로 한 번 더 정리하는 것입니다.

---

## 1) 전용 명단 탭 만들기

마스터 시트에 새 탭을 추가하고 이름을 **`포털명단`** 으로 합니다.
**1행에 아래 헤더**를 그대로 입력하세요(앱이 이 헤더명으로 컬럼을 인식합니다):

```
회원번호 | 성명 | 이메일 | fKF등급 | 민간 자격 | PG | 지역 | 소속 | Specialty | 생년월일 | 성별 | 2025 세미나 시수 | 2025 프로보노 시수 | 비고
```

- 헤더명은 `members.py`의 `ROSTER_HEADER_MAP` 키와 일치하면 됩니다(동의어 다수 허용).
- 연도별 실적 컬럼은 `2025 세미나 시수`처럼 **4자리 연도 + (세미나|프로보노|사회공헌|프로젝트)** 패턴이면 자동 인식됩니다. 연도를 늘리려면 컬럼만 추가하세요.
- 앱은 헤더 행을 자동 탐지하므로, 1행 위에 제목 행이 한 줄 있어도 무방합니다.

## 2) 행 채우기 — 두 가지 방법

### 방법 A (권장, 가장 간단): 기존 깔끔한 탭에서 가져오기
마스터에 이미 정리된 `*_print` 류 탭(예: `2022_print`)이 최신 데이터를 담고
있다면, `포털명단` A1에 한 줄 수식으로 통째로 끌어옵니다:

```
=QUERY('2022_print'!A:N, "select * where Col1 is not null", 1)
```

> 최신 연도 print 탭이 있으면 탭 이름/범위를 그쪽으로 바꾸세요. 컬럼 순서가
> 위 헤더와 다르면 `select Col1, Col2, ...` 로 순서를 맞춰 주세요.

### 방법 B: MDB에서 직접 끌어오기
`포털명단` 1행에 위 헤더를 입력하고, 2행부터 각 컬럼을 `MDB`에서 참조합니다.
회원 마스터는 `MDB!B21:B317`(회원번호) 범위입니다. 예:

```
A2: =ARRAYFORMULA(MDB!B21:B317)     // 회원번호
B2: =ARRAYFORMULA(MDB!C21:C317)     // 성명
D2: =ARRAYFORMULA(MDB!X21:X317)     // fKF등급
E2: =ARRAYFORMULA(MDB!Z21:Z317)     // 민간 자격
...                                  // 이메일/지역/소속/Specialty/생년월일/성별/시수는
                                     // MDB의 해당 열로 같은 방식 지정
```

> 이메일·지역·소속·Specialty·생년월일·성별·시수의 정확한 MDB 열은 시트
> 관리자가 가장 잘 아시므로, 해당 열로 매핑해 주세요. 헤더명만 맞으면 앱은
> 어느 열에 있든 인식합니다.

## 3) 서비스 계정에 읽기 권한 부여

포털 앱은 Google 서비스 계정으로 마스터 시트를 **직접 읽습니다**(IMPORTRANGE 아님).

1. Google Cloud에서 서비스 계정 생성 → JSON 키 발급, Sheets/Drive API 활성화
2. **마스터 시트를 서비스 계정 이메일(`...@...iam.gserviceaccount.com`)에 ‘뷰어’로 공유**
3. 배포 환경 변수 설정:
   - `GOOGLE_CREDENTIALS` = 서비스 계정 JSON 전체 문자열 (또는 `credentials.json` 파일 배치)
   - `FKF_MASTER_SHEET_ID` = `112_MplXH5gCAKODQyDcvymu8GsWav180zqcmq187jCQ`
   - `FKF_ROSTER_TAB` = `포털명단`
   - `SECRET_KEY` = 임의의 긴 문자열

## 4) 로그인 동작

- 회원은 **이름(성명) + 회원번호**(예: `fKF-25-1546`)로 로그인합니다.
- 회원번호로 행을 찾은 뒤 성명을 2차 확인 요소로 대조합니다.
- 로그인 성공 시 본인의 자격등급(fKF/민간/PG) · 교육 이수 · 연도별 실적을
  마이페이지 대시보드에서 조회합니다.

## 보안 메모

- 실제 회원 PII(이름·이메일·생년월일 등)는 **저장소에 절대 커밋하지 않습니다.**
  앱이 런타임에 서비스 계정으로만 조회합니다.
- 환경 변수/`credentials.json`은 배포 플랫폼 비밀로 관리하세요(`.gitignore` 처리됨).

## 실적 세부(records) 안내

현재 대시보드의 "실적 세부사항"은 전용 명단 탭에 실적 상세 컬럼이 없으면
비어 있습니다. 회원별 교육/프로젝트 상세 내역(날짜·교육명·프로젝트명·역할)을
표시하려면 별도 실적 탭(회원번호로 필터 가능한 평평한 표)을 추가하고 알려
주시면 `SheetsRepository`에 연결하겠습니다.
