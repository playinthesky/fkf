## 1. 한 문장 정의

> **시민·전문가·행정이 충분한 정보를 바탕으로 깊이 있게 토론하여 합의에 이르는 과정(숙의·熟議)을,
> AI(Claude)와 사람이 함께 설계·진행·분석하는 컨설팅 서비스.**

우리는 이것을 **AX(AI + Human) 방식의 숙의형 데이터 분석**이라 부르며, 모든 프로젝트에 핵심으로 내장합니다.

---

## 2. 왜 하는가 (문제의식)

| 기존 공론화의 한계 | AX 숙의가 해결하는 방식 |
|---|---|
| 진행자 역량에 품질이 좌우됨 | AI 실시간 요약·논점 추출로 진행 표준화 |
| 발언 쏠림·소수의견 누락 | 발언 균형 지표(지니계수) 모니터링 |
| 결과 분석에 수주가 소요 | Claude 기반 자동 분석으로 즉시 구조화 |
| 재현·검증이 어려움 | 표준 프로세스 + 품질지수(DQI)로 재현 가능 |

---

## 3. 무엇을 제공하는가 (서비스 구성)

```
[고객 프로젝트]
├─ 솔루션: 숙의형 데이터 분석 (정성분석 + 구조적 토픽모델링 STM)  ← 항상 포함
├─ 방식 : AX (Human + AI 협업)                                    ← 항상 포함
└─ 의제 : 참여예산 / 도시계획 / AI윤리 / 지역현안 ...             ← 프로젝트별 다름
```

- **적용 분야**: 공공정책(예산·조례·도시계획), 기술·윤리(AI윤리·데이터 거버넌스), 지역현안(환경·교통·복지)

---

## 4. 어떻게 진행하는가 (표준 5단계)

```
[1. 준비]──▶[2. 학습]──▶[3. 숙의]──▶[4. 합의]──▶[5. 환류]
 2~4주       1~2주       2~4주       1주        지속
```

| 단계 | 핵심 활동 | 대표 산출물 |
|------|-----------|-------------|
| **준비** | 의제 확정·참여자 모집·AX 도구 세팅 | 숙의 의제서, 학습자료 |
| **학습** | 사전학습·전문가 브리핑·Q&A | 학습 패키지, 참석 확인 |
| **숙의** | 소그룹·패널·교차·전체 토론 | 회의록, AI 실시간 요약 |
| **합의** | 합의안 작성·숙의투표·확정 | 최종 합의문 |
| **환류** | 결과 공개·이행 모니터링·평가 | 공개 보고서, 평가보고서 |

---

## 5. AI는 정확히 어디에 들어가는가

| AI 지원 기능 | 설명 | 적용 단계 |
|---|---|---|
| 자동 요약 | 토론 내용 실시간 요약 | 숙의 |
| 논점 추출 | 핵심 쟁점 구조화 | 학습·숙의 |
| 감정/반응 분석 | 참여자 반응 모니터링 | 숙의 |
| 팩트체크 | 발언 근거 검증 | 전 단계 |
| 합의 시뮬레이션 | 투표 결과 예측 | 합의 |

> **사람의 자리**: 중립적 퍼실리테이션, 윤리·법률 검토, 최종 의사결정은 사람이 담당합니다.
> AI는 진행을 **돕는 도구**이지 결정 주체가 아닙니다.

---

## 6. 품질을 어떻게 보장하는가

**숙의 품질지수(DQI, Deliberation Quality Index)** 로 측정·인증합니다.

| 지표 | 목표치 |
|---|---|
| 참여율 | ≥ 90% |
| 발언 균형(지니계수) | ≤ 0.3 |
| 근거 활용(팩트 인용률) | ≥ 60% |
| 상호 존중(부정표현 비율) | ≤ 5% |
| 합의 수준(최종 동의율) | ≥ 70% |

- 인증 등급: **Bronze → Silver → Gold → Platinum**

---

## 7. 우리의 차별점 (한 장 요약)

1. **숙의 × 데이터분석 결합** — 토론을 정성·정량(STM) 데이터로 전환해 근거 있는 결론 도출
2. **AX 표준 프로세스** — 진행자 역량 의존도를 낮추고 재현 가능한 품질 확보
3. **AI 실시간 지원** — 요약·논점·팩트체크로 숙의의 밀도를 높임
4. **검증 가능한 품질지표(DQI)** — "잘 됐다"가 아니라 숫자로 증명
