# 마스터 일정 생성 규칙

> Version 0.1.0 | 2026-06-04

봇이 사용자가 제공한 (킥오프일, 종료일, 숙의 본 회차 수)를 받아 자동으로 마스터 일정표를 산출하는 규칙.

---

## 1. 입력 파라미터

| 파라미터 | 예시 | 출처 |
|---------|------|------|
| `kickoff_date` | 2026-06-08 (월) | 사용자 응답 또는 계약서 |
| `end_date` | 2026-12-31 | 계약서 |
| `deliberation_rounds` | 1 | 사용자 응답 (기본 1, 표준은 4) |
| `participant_count` | 30~50 | 자료 또는 추정 |
| `committee_meeting_freq` | 격주 또는 월1회 | 사용자 응답 |

---

## 2. 단계별 기간 배분 규칙

총 기간 = `end_date` − `kickoff_date` (주 단위 환산)

### 2.1 표준 배분 (총 기간이 표준 16~20주 범위일 때)
| 단계 | 비율 | 비고 |
|------|------|------|
| Stage 1 준비 | 35% | 의제 선정·자료 준비 |
| Stage 2 학습 | 10% | 자료 배포·OT |
| Stage 3 숙의 | 25% | `deliberation_rounds` × 1~2주 |
| Stage 4 합의 | 10% | 1~2주 |
| Stage 5 환류 | 20% | 결과보고·백서 |

### 2.2 장기 배분 (총 기간 25주 이상, 예: 6/8~12/31)
숙의 회차가 적을 경우 (1회):
| 단계 | 기간 | 비고 |
|------|------|------|
| Stage 1 준비 | 14주 | 사전 미팅 다수 흡수 |
| Stage 2 학습 | 3주 | |
| Stage 3 숙의 | 1주 (1회) | |
| Stage 4 합의 | 2주 | |
| Stage 5 환류 | 8주 이상 | 백서·결과보고회 |

---

## 3. 트랙별 시작·종료 규칙

| 트랙 | 시작 | 종료 |
|------|------|------|
| 1. 일정/거버넌스 | W1 (킥오프) | 종료 |
| 2. 홍보/소통 PR-1 | W2 | 의제확정 직전 |
| 2. 홍보/소통 PR-2 | 의제확정 직후 | 숙의 시작 |
| 2. 홍보/소통 PR-3 | 숙의 시작 | 숙의 종료 |
| 2. 홍보/소통 PR-4 | 합의 직후 | 환류 종료 |
| 3. 숙의 품질 (AI-bdl 셋업) | W3~ | 학습 단계 종료 전 가동 완료 |
| 3. 숙의 품질 (NotebookLM 셋업) | W3~ | 학습 단계 시작 시 가동 |
| 3. 숙의 품질 (Meetup 운영) | 학습 시작 | 숙의 종료 |
| 4. 참여단 모집 | 의제 확정 후 | OT 1주 전 |
| 4. 참여단 메시지 발송 | 합의 직후 | 합의 + 1주 |
| 5. 안전 (SAFETY CODE) | 모집 시작 시 점검 | 모든 현장 회차 종료까지 |
| 6. 운영 SOP (회의록) | 킥오프 첫 회의부터 | 마지막 회의까지 |
| 6. 운영 SOP (결과보고회) | 환류 단계 중 | |
| 7. 산출물 품질 (백서) | 결과 보고서 직후 | 종료 |

## 4. 크리티컬 패스 자동 마킹

봇이 자동으로 다음 항목을 🔴 크리티컬로 표시:

| 항목 | 이유 |
|------|------|
| 의제 확정 | 모든 후속 활동의 전제 |
| 참여단 모집 마감 | 가족 필터 사전 정의 필요 |
| SAFETY CODE 점검 | 보험 누락 = 작년 사고 반복 |
| AI-bdl 첫 가동 (숙의 직전) | 사전 시뮬레이션 필수 |
| 소통광장 아이디 확보 | PR 1단계 의존 |

---

## 5. 출력 형식

봇이 생성하는 일정표는 다음 두 가지 형태로:

### 5.1 주차별 테이블 (마크다운)
```markdown
| 주차 | 일자 | 단계 | 핵심 활동 | 산출물 |
| W1 | 6/8~6/14 | 준비 | 킥오프 + 추진위 1차 | 추진위 회의록 |
| ... | ... | ... | ... | ... |
```

### 5.2 간트차트 (PPT 슬라이드 1장)
- python-pptx로 직접 그림
- 색상: AX Blue (단계), AX Teal (트랙), AX Gold (마일스톤)
- 4:3 슬라이드 크기에 맞춰 자동 스케일
