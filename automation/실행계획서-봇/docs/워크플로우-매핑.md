## 1. 매핑 원리

```
┌─────────────────────┐     ┌────────────────────┐     ┌──────────────────┐
│ 스픽스 워크플로우     │  ×  │ 7트랙 (활동 카테고리) │  ×  │ 프로젝트별 11이슈  │
│ (5단계 표준)         │     │                    │     │ (예: 서울교육공론) │
└─────────────────────┘     └────────────────────┘     └──────────────────┘
        │
        ▼
   각 단계에서 어느 트랙의 어느 액션을 수행할지 매트릭스
```

## 4. 봇 활용 방식

봇이 새 프로젝트 상담 시 이 매트릭스를 기준으로:

1. **입력 자료에서 11이슈에 해당하는 키워드 추출** → 어떤 ⚠️ 항목이 이 프로젝트에도 해당하는지 자동 판단
2. **누락된 트랙·액션을 사용자에게 질문** ("작년 보험 미가입 이슈 있었나요?")
3. **표준 액션은 기본 포함** — 의제 확정·자료집·OT·합의 등
4. **트랙별 시작·종료 시점은 일정-템플릿.md 규칙으로 자동 산출**
