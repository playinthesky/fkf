## 1. 컨셉

### 1.1 한 줄
Slack `/실행계획서` 슬래시 커맨드 → **상담형 AI 봇**이 입력 자료를 받아 **트랙별 실행계획 + 마스터 일정표**를 합성하고 **PPT(스픽스형) + HWPX**로 산출.

### 1.2 핵심 원칙
1. **상담형**: 한 번에 뱉지 않고 대화로 빈 곳 채움
2. **표준 내장**: 스픽스 숙의 5단계 워크플로우를 항상 기본 뼈대로 사용
3. **AX 차별화 명시**: 모든 산출물에 (a) 숙의형 데이터 분석(STM) (b) AX(Human+AI) 협업 명시
4. **재사용 가능**: 결과물은 GitHub 레포 `projects/[프로젝트명]/`에 자동 보관
5. **임의 추정 금지** ⭐: 입력 자료에 없는 정보는 추측 ❌. 사용자에 질문하거나 "가정"·"협의 필요"로 명시. 휴먼 컨설턴트가 클라이언트와 협의해 확정할 부분.
6. **연도별 사례 일반화 금지** ⭐: 작년 사례는 참고일 뿐. 특정 연도가 비정상년(예: 2025 = 내란)일 수 있으니 단순 매핑 ❌. 여러 연도 비교 또는 사용자 확인.
7. **마스터 산출물 = HTML**: 실행계획서의 source of truth는 HTML(v0.x.y). HWPX·PPT는 HTML에서 파생된 변환물.

### 1.3 4총사 호칭 (회사 호칭)
| AI | 호칭 | 담당 |
|----|------|------|
| Claude (코드/터미널) | **본승지** | 슬랙·이 봇 자체·실행계획서 HTML 마스터 작성 |
| Claude (모바일) | 모별감 | 모바일 응답 |

| Antigravity | **구편수** | 자동화 인프라·신규 레포 셋업 |

## 3. 입력 처리

### 3.1 지원 파일 포맷
| 포맷 | 추출 방법 | 라이브러리 |
|------|----------|-----------|
| PDF (텍스트) | 텍스트 추출 | `pdfplumber`, `PyMuPDF` |
| PDF (스캔/이미지) | OCR | `pytesseract` + `pdf2image` (한국어 모드) |
| HWP | 텍스트 추출 | `pyhwp` 또는 `olefile` |
| HWPX | XML 파싱 | `lxml` (HWPX는 ZIP+XML) |
| DOCX | 텍스트 추출 | `python-docx` |
| XLSX | 시트 추출 | `openpyxl` |
| MD/TXT | 직접 읽기 | stdlib |
| 이미지 | OCR | `pytesseract` |

### 3.2 입력 소스
1. **Slack 첨부파일** — `file_shared` 이벤트 픽업
2. **Drive 최근 파일** — 사장님 다운로드 폴더 → Drive 자동 동기화 활용
3. **Drive 링크 직접 붙여넣기**
4. **fkf 레포 표준 솔루션** — 항상 자동 포함 (트랙3 AX-bdl, NotebookLM 등)

## 4. 상담 엔진

### 4.1 시스템 프롬프트 골자
```
당신은 코리아스픽스의 AX 컨설팅 비서(본승지)입니다.
직원이 제공한 입력 자료(제안서·RFP·기술협상결과·계약서·업무분장 등)와
fkf 레포의 클립북(`projects/[프로젝트명]/06_클립북/`)을 종합해
스픽스 숙의 컨설팅 표준 워크플로우(5단계)에 매핑한
실행계획서 HTML(v0.x.y)을 함께 만드는 상담을 진행합니다.

원칙:
1. 자료에 없는 정보는 **추측하지 말고 사용자에게 질문**
2. 추정이 필요하면 "가정" 또는 "협의 필요"로 명시
3. **연도별 사례 일반화 금지** — 작년이 비정상년일 수 있음(예: 2025=내란).
   여러 연도 비교 또는 사용자 확인.
4. 항상 다음을 산출물에 포함:
   - 숙의형 데이터 분석(STM)
   - AX(Human+AI) 방식
   - SAFETY CODE (안전 트랙)
   - 4단계 홍보 계획
5. 기술협상 이슈는 트랙별로 빠짐없이 반영
6. **마스터 산출물은 HTML** — HWPX·PPT는 별동수가 변환 레포로 처리
7. 클라이언트와 협의해야 할 항목은 `05_협의필요_항목.md`로 별도 정리
8. CLAUDE.md 작업 규칙 준수 (정부 보고용 PPT 4:3 등)
```

## 5. 산출물 생성

### 5.0 산출물 변환 워크플로우 (4총사 분담)

```
[본승지] HTML v0.x.y 마스터 생성
        │
        ├─── /실행계획서 변환 hwpx ──▶ [별동수] 한글 공문서 변환 레포
        │                              레포명: TBD ← 사용자 확인 필요
        │                              산출: .hwpx
        │
        └─── /실행계획서 변환 ppt ──▶ [별동수] PPT 변환 레포
                                       레포명: TBD ← 사용자 확인 필요 / 없으면 [구편수]가 신설
                                       적용: 슬라이드_디자인_지침서 v0.1.0 강제
                                       산출: .pptx (4:3, AX 컬러, Pretendard)
```

변환 레포가 갖춰야 할 기본 요건:
- **입력**: HTML 또는 마크다운
- **출력**: 단일 .hwpx 또는 .pptx
- **자동화**: GitHub Action으로 PR 트리거 시 자동 변환 가능해야
- **스타일 강제**: PPT는 `templates/슬라이드_디자인_지침서_v0.1.0.md` 준수

### 5.1 PPT (스픽스형 4:3)
- **규격**: 10" × 7.5" (정부 보고용)
- **마스터 슬라이드**: `templates/슬라이드_디자인_지침서_v0.1.0.md` 준수
- **컬러**: AX Blue `#1E3A5F` / AX Teal `#2A9D8F` / AX Gold `#E9C46A`
- **폰트**: Pretendard (대체: 맑은 고딕)
- **그래픽 자산**: `assets/ax-kit/*.png` (사전 생성 — 헤더바·구분선·아이소 블록·아이콘 등)
- **표준 슬라이드 구성**:
  1. 표지
  2. 프로젝트 개요 (의제·기간·예산·조직)
  3. AX 컨설팅 접근법 (STM + Human+AI 명시)
  4. 7트랙 구조도
  5~11. 트랙별 1장씩 (활동·산출물·담당)
  12. 마스터 일정표 (간트)
  13. 크리티컬 패스 + 리스크
  14. 산출물 목록
  15. 백 커버

### 5.2 HWPX
- Pandoc Markdown → DOCX → 한컴 자동변환 (가장 안정적)
- 또는 `hwpxlib` 직접 작성 (표·서식 한계)

### 5.3 저장 위치
```
fkf/projects/[프로젝트명]/
├─ README.md
├─ 01_입력자료_요약.md         ← 추출된 텍스트 요약
├─ 02_트랙별_정리.md            ← 7트랙 합성 결과
├─ 03_마스터_일정표.md
├─ 실행계획서_v[N].pptx
├─ 실행계획서_v[N].hwpx
└─ session_log.md               ← 상담 대화 기록
```

## 6. 7트랙 (스픽스 표준 + 2026 서울교육공론화 11이슈 반영)

| 트랙 | 핵심 활동 | 워크플로우 단계 |
|------|----------|----------------|
| 1. 일정/거버넌스 | 추진위 운영, 의제선정위, 마일스톤 | 전 단계 |
| 2. 홍보/소통 | 4단계 PR(소개→의제선정→과정→결과), 소통광장, 쇼츠·인포그래픽 | 준비~환류 |
| 3. 숙의 품질 | AI-bdl(특허 출원중), NotebookLM Meetup, 테이블 설계, 퍼실 교육 | 학습·숙의 |
| 4. 참여단 관리 | 가족 비중 필터, 1:1 결과 메시지, 교육감 땡큐 메시지 | 준비·환류 |
| 5. 안전 (SAFETY CODE) | 보험·증서 제출, 안전요원, 지정병원, 응급·주차 | 숙의 회차 전 |
| 6. 운영 SOP | 회의록(녹취+앱), 결과보고회 참가·촬영, 카운터파트너 | 전 단계 |
| 7. 산출물 품질 | 백서 가독성(글로벌팀), DQI 측정, 검수 체크리스트 | 환류 |
