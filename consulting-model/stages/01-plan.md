# 1단계: PLAN (수주)

> 프로젝트의 시작. 클라이언트와 만나 계약까지.

---

## 목표

- 클라이언트 니즈 파악
- 사업 범위·예산·일정 합의
- 계약 체결

---

## 기간

1~4주 (규모에 따라)

---

## 핵심 질문

| 질문 | 왜 중요한가 |
|------|------------|
| 누가 발주하나? | 진짜 의사결정자 파악 |
| 왜 필요한가? | 표면 vs 실제 니즈 |
| 예산은? | 가능한 범위 확인 |
| 일정은? | 리소스 배분 |
| 기대 결과는? | 성공 기준 명확화 |

---

## 세부 단계

### 1.1 초기 접촉
- 문의 접수 (전화/이메일/소개)
- 1차 상담 일정 잡기
- 기본 정보 수집

**체크리스트:**
- [ ] 문의 내용 슬랙 기록 (#client-프로젝트명)
- [ ] 발주처 기본 정보 확인
- [ ] 1차 상담 일정 확정

### 1.2 니즈 파악
- 1차 상담 진행
- 클라이언트 배경·목적 청취
- 숨겨진 니즈 발굴

**체크리스트:**
- [ ] 상담 기록 작성
- [ ] 표면 니즈 vs 실제 니즈 구분
- [ ] 의사결정 구조 파악

### 1.3 제안서 작성
- 사업 범위 정의
- 일정·예산 산정
- 제안서 작성

**체크리스트:**
- [ ] 사업 범위 명확화
- [ ] 예산 산출 (인건비/운영비/시스템)
- [ ] 일정표 작성
- [ ] 제안서 초안 → 대표 검토

### 1.4 협상·계약
- 제안 발표
- 협상 (범위/예산/일정 조정)
- 계약 체결

**체크리스트:**
- [ ] 제안 발표 완료
- [ ] 수정 요청 반영
- [ ] 계약서 검토 (법률)
- [ ] 계약 체결 → 착수금 수령

| 대표 | 제안서 검토, 계약 승인 |
| Claude | 제안서 초안 보조, 예산 검토 |

---

## 주의사항

### ⚠️ 흔한 실수
1. **발주처 오인** — 본청 vs 산하기관 구분
2. **예산 저평가** — 인건비·간접비 누락
3. **범위 모호** — "이것도 해주세요" 방지
4. **의사결정자 미파악** — 실무자만 만남

### ✅ 좋은 습관
1. 첫 상담에서 의사결정 구조 파악
2. 제안서에 "포함/미포함" 명시
3. 계약 전 착수금 조건 확인
4. 모든 상담 내용 슬랙 기록
