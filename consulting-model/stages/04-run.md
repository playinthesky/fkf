# 4단계: RUN (실행)

> 본 게임. 숙의 현장에서 일어나는 모든 것.

---

## 목표

- 숙의 프로세스 원활 진행
- 참여자 의견 수렴
- 실시간 이슈 대응

---

## 기간

4~12주 (프로젝트 규모에 따라)

---

## 핵심 질문

| 질문 | 왜 중요한가 |
|------|------------|
| 참여자가 이해하고 있나? | 숙의 품질 |
| 모든 의견이 나오고 있나? | 균형 |
| 일정대로 진행되나? | 효율 |
| 돌발 상황은? | 대응력 |
| 기록되고 있나? | 결과물 |

---

## 세부 단계

### 4.1 오리엔테이션
- 참여자 등록·접수
- 전체 안내
- 아이스브레이킹

**체크리스트:**
- [ ] 참석 체크 (출석부)
- [ ] 명찰 배부
- [ ] 전체 일정·규칙 안내
- [ ] 참여 동의서 서명
- [ ] 아이스브레이킹 진행

### 4.2 학습 세션
- 배경 자료 공유
- 전문가 발표
- Q&A

**체크리스트:**
- [ ] 학습자료 배부 확인
- [ ] 발표자 소개·진행
- [ ] 시간 관리 (발표 + Q&A)
- [ ] 질문 기록
- [ ] 이해도 체크

### 4.3 숙의 세션
- 소그룹 토론
- 전체 토론
- 의견 수렴

**체크리스트:**
- [ ] 소그룹 배정 (무작위/혼합)
- [ ] 퍼실리테이터 배치
- [ ] 토론 규칙 안내
- [ ] 시간 관리
- [ ] 의견 기록 (포스트잇/AI BDL)
- [ ] 중간 정리·공유

### 4.4 의견 수렴·투표
- 쟁점별 의견 정리
- 투표/합의 도출
- 결과 공유

**체크리스트:**
- [ ] 쟁점별 선택지 정리
- [ ] 투표 방식 안내 (스티커/전자)
- [ ] 투표 진행
- [ ] 결과 집계·발표
- [ ] 소수 의견 기록

### 4.5 마무리·환류
- 결과 요약
- 참여자 피드백
- 감사 인사

**체크리스트:**
- [ ] 전체 결과 요약 발표
- [ ] 향후 일정 안내
- [ ] 참여자 만족도 조사
- [ ] 참여 수당 지급
- [ ] 단체 사진 (동의 시)

---

## 현장 운영 팁

### 퍼실리테이션 원칙

| 원칙 | 설명 |
|------|------|
| 중립 유지 | 개인 의견 배제 |
| 균등 발언 | 한 사람이 독점 안 되게 |
| 경청 | 끝까지 듣고 정리 |
| 시간 관리 | 타이머 필수 |
| 기록 | 모든 의견 기록 |

### 돌발 상황 대응

| 상황 | 대응 |
|------|------|
| 참여자 갈등 | 휴식 제안 / 개별 대화 |
| 발언 독점 | "다른 분 의견도" 유도 |
| 침묵 | 소그룹 먼저 / 포스트잇 활용 |
| 시간 초과 | 핵심 쟁점 우선 / 연장 협의 |
| 장비 문제 | 백업 전환 / 아날로그 |

## 주의사항

### ⚠️ 흔한 실수
1. **시간 초과** — 토론 시간 관리 실패
2. **기록 누락** — 중요 의견 놓침
3. **발언 불균형** — 일부만 발언
4. **피드백 생략** — 참여자 의견 미수렴

### ✅ 좋은 습관
1. 타이머 항상 사용
2. 포스트잇으로 전원 의견 수렴
3. 중간 정리 시간 확보
4. 매일 슬랙 일일 보고
