- 발주처 전달·마무리

---

## 기간

2~4주

## 세부 단계

### 5.1 결과 분석
- 토론 기록 정리
- 투표 결과 분석
- 참여자 피드백 분석

**체크리스트:**
- [ ] 토론 기록 통합·정리
- [ ] 투표 결과 통계 분석
- [ ] 참여자 만족도 분석
- [ ] 핵심 발견점 도출
- [ ] 소수 의견 정리

### 5.2 결과보고서 작성
- 초안 작성
- 내부 검토
- 최종본 완성

**체크리스트:**
- [ ] 보고서 구조 설계
- [ ] 초안 작성 (Claude 보조)
- [ ] 시각자료 제작 (그래프/도표)
- [ ] 대표 검토
- [ ] 수정·최종본 완성
- [ ] HWPX 변환

### 5.3 발주처 전달
- 결과 발표
- 피드백 수렴
- 정산·마무리

**체크리스트:**
- [ ] 결과 발표 일정 협의
- [ ] 발표 진행
- [ ] 발주처 피드백 청취
- [ ] 수정 요청 반영
- [ ] 최종 결과물 전달
- [ ] 정산 (잔금 수령)
- [ ] 완료 확인서

### 5.4 내부 회고
- 프로젝트 회고
- 잘한 점 / 아쉬운 점
- 개선안 도출

**체크리스트:**
- [ ] 팀 회고 미팅
- [ ] 잘한 점 3가지
- [ ] 아쉬운 점 3가지
- [ ] 다음 프로젝트 개선안
- [ ] 회고 기록 슬랙 공유

### 5.5 모델 업데이트
- 교훈 → 컨설팅 모델 반영
- 템플릿 개선
- 사례 아카이브

**체크리스트:**
- [ ] 새로 배운 패턴 정리
- [ ] 컨설팅 모델 PR 생성
- [ ] 템플릿 개선안 반영
- [ ] 사례집 업데이트
- [ ] 대표 승인 → 머지

## 결과보고서 구조 (표준)

```
1. 개요
   - 사업명, 기간, 발주처
   - 목적, 의제

2. 추진 경과
   - 일정, 참여자, 진행 과정

3. 숙의 결과
   - 쟁점별 논의 내용
   - 투표/합의 결과
   - 주요 의견

4. 참여자 분석
   - 참여 현황
   - 만족도
   - 피드백

5. 결론 및 제언
   - 핵심 결론
   - 정책/서비스 제언
   - 후속 과제

6. 부록
   - 세부 데이터
   - 참고자료
```

## 주의사항

### ⚠️ 흔한 실수
1. **보고서 지연** — 마감 후 방치
2. **회고 생략** — 바빠서 스킵
3. **교훈 미반영** — 같은 실수 반복
4. **아카이브 누락** — 사례 소실

### ✅ 좋은 습관
1. 실행 종료 후 2주 내 보고서 완료
2. 회고는 기억이 생생할 때 (1주 내)
3. 모든 교훈 → 컨설팅 모델 PR
4. 사례집 꾸준히 축적

## 후속 관리

### 발주처 관계 유지
- 완료 후 1개월: 후속 연락
- 6개월: 결과 활용 현황 확인
- 매년: 새 사업 기회 탐색

### 팀 성장
- 프로젝트별 역량 평가
- 교육 필요 영역 파악
- 다음 프로젝트 역할 조정
