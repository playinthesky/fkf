# 2단계: BUILD (설계)

> 어떻게 할 것인가. 실행 전 모든 것을 설계.

---

## 목표

- 실행계획서 완성
- 참여자·퍼실리테이터 확정
- 자료·도구 준비

---

## 기간

2~4주

---

## 핵심 질문

| 질문 | 왜 중요한가 |
|------|------------|
| 의제가 무엇인가? | 숙의 주제 명확화 |
| 누가 참여하나? | 참여자 구성 설계 |
| 어떤 방식으로? | 숙의 형식 결정 |
| 무엇이 필요한가? | 자료·도구·장소 |
| 어떻게 평가하나? | 성공 기준 설정 |

---

## 세부 단계

### 2.1 의제 설계
- 숙의 주제 구체화
- 쟁점 구조화
- 선택지 정리

**체크리스트:**
- [ ] 핵심 의제 1문장 정의
- [ ] 하위 쟁점 3~5개 도출
- [ ] 각 쟁점별 선택지 정리
- [ ] 발주처 의제 확인·승인

### 2.2 참여자 설계
- 참여자 구성 기준 설정
- 모집 방법 결정
- 규모 확정

**체크리스트:**
- [ ] 참여자 유형 정의 (시민/전문가/이해관계자)
- [ ] 모집 인원 확정
- [ ] 모집 방법 결정 (무작위/공모/추천)
- [ ] 참여 수당·인센티브 설계

### 2.3 프로세스 설계
- 숙의 형식 결정
- 일정표 작성
- 퍼실리테이션 설계

**체크리스트:**
- [ ] 숙의 형식 결정 (원탁/소그룹/온라인)
- [ ] 전체 일정표 작성
- [ ] 세션별 상세 타임라인
- [ ] 퍼실리테이터 배치 계획

### 2.4 자료 준비
- 학습자료 제작
- 발표자료 준비
- 도구·시스템 점검

**체크리스트:**
- [ ] 참여자용 학습자료 작성
- [ ] 배경자료 수집·정리
- [ ] 발표용 PPT 제작
- [ ] 투표/의견수렴 도구 준비
- [ ] AI BDL 시스템 점검

### 2.5 실행계획서 완성
- 전체 계획 문서화
- 발주처 승인
- 킥오프 준비

**체크리스트:**
- [ ] 실행계획서 초안 작성
- [ ] 내부 검토 (대표)
- [ ] 발주처 제출·피드백
- [ ] 최종 승인

## 주의사항

### ⚠️ 흔한 실수
1. **의제 모호** — "농업 발전"처럼 너무 넓음
2. **참여자 편향** — 특정 집단만 모집
3. **일정 과소평가** — 준비 시간 부족
4. **자료 과다** — 참여자가 소화 못함

### ✅ 좋은 습관
1. 의제를 질문형으로 ("~해야 하는가?")
2. 참여자 다양성 체크 (성별/연령/지역)
3. 여유 일정 확보 (버퍼 20%)
4. 학습자료 10페이지 이내
