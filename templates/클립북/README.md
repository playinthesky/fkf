# 클립북 (Clipbook) 표준 템플릿

> 새 프로젝트가 시작될 때 자동 생성되는 "사전 자료 지형도"

---

## 클립북이란

특정 주제(공론화 의제·정책 등)에 대해 **이미 세상에 존재하는 자료들을 매체별로 모은 사전 조사 지형도**.

- 자료맵(어디에 무엇이 있다)이 아니라 **실제 클립**(보도자료·언론기사·영상·연구보고서)을 직접 모음
- 실행계획서 작성 시 **참고 자료의 1차 입력**으로 사용
- 봇이 자동으로 gaps를 채우며 점진적으로 풍부해짐

---

## 파일 구조

| 파일 | 용도 | 누가 보나 |
|------|------|----------|
| `*.html` | 사람이 보는 클립북 (디자인 입힌 보고서) | 컨설턴트·클라이언트 |
| `*-handoff.md` | 봇이 보는 데이터 + 작업 지시 | 본승지·별동수·구편수 |

handoff.md 안에는:
- 마크다운 헤더 + 봇 작업 지시 (gaps 채우기)
- JSON 데이터 (machine-readable)

---

## 채널(channel) 구조 — 1차축은 "매체"

| ID | 매체 | 예시 |
|----|------|------|
| A | 공식 보도 (발주처 enews) | 교육청 공식 보도자료 |
| B | 외부 언론 보도 | 신문·방송 미디어 클립 |
| C | 누리집 참여·의제 채널 | 시민 제안·정책 설문 |
| D | 영상·카드뉴스·SNS | YouTube·Instagram 등 |
| E | 보고서·연구·웹진 | KCI 학술논문·이슈페이퍼 |

채널은 프로젝트 성격에 따라 가감 가능.

---

## 클립 상태 코드

| 코드 | 의미 | 봇 액션 |
|------|------|--------|
| `verified` | URL·존재 확인된 실제 클립 | 그대로 유지 |
| `channel_only` | 채널 진입점만 확인, 전용 클립 미특정 | gap 후속 |
| `index` | 목록/검색 페이지 (개별 클립 아님) | 봇이 검색해서 verified로 승격 |
| `gap` | URL 없음 — **봇이 채울 대상** | 1순위 작업 |

## 자동화 워크플로우

```
[새 프로젝트 시작]
       │
       ▼
[본승지(저)] 클립북 v0.1.0 초안 생성
   - 주제 분석
   - 웹/Drive 검색 → 매체별 클립 수집
   - handoff.md JSON 작성 + gaps 명시
       │
       ▼
[별동수(Codex)] gaps 자동 채우기
   - YouTube/SNS 개별 URL 검색
   - 200 응답 검증
   - 새 클립 추가, doc_version 패치 상승
       │
       ▼
[구편수(안티그래비티)] 인프라 자동화
   - 매주 자동 갱신
   - 신규 보도자료 자동 감지
       │
       ▼
[실행계획서 봇] 클립북을 입력 자료로 활용
```

## 사용법

### 새 프로젝트에 클립북 만들기
```bash
cp templates/클립북/클립북_템플릿_v0.1.0.html \
   projects/[프로젝트명]/06_클립북/[주제]_클립북_v0.1.0.html

cp templates/클립북/클립북_handoff_템플릿_v0.1.0.md \
   projects/[프로젝트명]/06_클립북/[주제]_클립북_handoff_v0.1.0.md
```

그 다음 채널·클립을 프로젝트 주제에 맞게 채움.

### 봇에게 자동 생성 의뢰 (구상)
```
/클립북 [프로젝트명] [주제]
```
→ 봇이 자동으로 채널별 검색·수집 → v0.1.0 초안 → 사용자 검토.
