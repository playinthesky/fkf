# AI시대 퍼실리테이터 — 클로드(Claude) 활용 양성과정 커리큘럼
> Version 0.1.0 | 2026-06-22 | AX 표준화 시리즈 · 교육/전환(D) 코드체계

---

## 0. 문서 목적

본 문서는 **fkf 회원**을 대상으로 하는 **「AI시대 퍼실리테이터 — 클로드 활용」 양성과정**의
표준 커리큘럼 설계서입니다. 과정 개설·운영·평가·자격 인증의 단일 기준으로 사용합니다.

- **솔루션 연계**: 모든 실습은 fkf의 핵심 기술인 **숙의형 데이터 분석(STM)** 과 연결됩니다.
- **방식**: 전 과정 **AX(Human + AI 협업)** 를 전제로 합니다. 퍼실리테이터가 Claude를
  "보조 진행자·기록자·분석가"로 활용하는 실전 역량을 길러 **수료 시 자격을 부여**합니다.

> 한 줄 요약: **숙의 퍼실리테이션 + Claude 실전 활용**을 결합해, 현장에서 바로 쓰는
> "AI 협업형 퍼실리테이터"를 양성하는 자격 과정.

## 1. 과정 개요

### 1.1 배경

오프라인 토론·숙의 현장에서 퍼실리테이터는 ① 중립적 진행, ② 발언 균형 관리,
③ 실시간 기록·요약, ④ 쟁점 구조화, ⑤ 결과 보고를 동시에 수행해야 합니다.
Claude를 활용하면 ③~⑤의 부담을 크게 줄이고, 퍼실리테이터가 ①②(사람 사이의 진행)에
집중할 수 있습니다. 본 과정은 이 **분업 구조(사람=진행/판단, AI=기록/분석/초안)** 를
체화시키는 데 목적이 있습니다.

### 1.2 양성 인재상

| 구분 | 정의 |
|------|------|
| 한 문장 | "Claude를 곁에 둔 숙의 퍼실리테이터" |
| 핵심 역량 | 중립적 토론 진행 + AI 기반 기록·요약·분석·보고 |
| 차별점 | 도구 사용법이 아니라 **숙의 품질(DQI)을 높이는 AI 활용**에 초점 |

### 1.3 학습 목표 (수료 시 도달 수준)

수료생은 다음을 **혼자서** 할 수 있어야 합니다.

1. 숙의/워크숍 의제를 받아 **진행 시나리오와 토론 설계**를 Claude와 함께 30분 내 초안화
2. 토론 현장에서 **실시간 요약·논점 추출**을 Claude로 보조 진행
3. 녹취·메모를 **회의록·쟁점 지도·합의안 초안**으로 자동 변환
4. 토론 데이터를 **숙의형 데이터 분석(STM)** 관점으로 1차 구조화
5. AI 활용 시 **중립성·개인정보·환각(hallucination) 리스크**를 통제
6. 결과물을 fkf 표준(보고서/슬라이드)에 맞춰 산출

### 1.4 대상 및 선발

| 항목 | 내용 |
|------|------|
| 대상 | fkf 회원 중 숙의·토론·워크숍 진행 경험자 또는 진행 예정자 |
| 사전 요건 | Claude.ai 계정(무료 가능, 실습 일부는 유료 기능 권장), 노트북 지참 |
| 정원 | 1기 **12~16명** (실습 밀착 지도를 위해 16명 상한) |
| 선발 | 지원서 + 간단 인터뷰(진행 경험·동기), AI 사전지식 불요 |

> **AI 입문자도 가능**하도록 Module 0(준비편)을 둡니다. 단, 본 과정은 *입문 강좌가 아니라
> 양성(자격) 과정*이므로 수료 기준을 충족해야 자격이 부여됩니다.

## 2. 과정 구조

### 2.1 전체 구성

```
[M0. 준비]─▶[M1. 기초]─▶[M2. 설계]─▶[M3. 현장]─▶[M4. 분석]─▶[M5. 윤리]─▶[M6. 인증]
  사전        클로드      토론설계     실시간      숙의데이터   리스크      캡스톤
  온라인      기본기      ×AI         진행보조     분석(STM)    통제        +평가
```

- **총 시수**: 32시간 (집합 24h + 온라인 사전/과제 8h)

- **방식**: 강의 30% · 실습 50% · 동료 피드백/리뷰 20%
- **도구**: Claude.ai(웹), Projects/Artifacts, (선택) Claude Code, fkf 표준 템플릿

## 3. 모듈별 상세

### M0. 준비편 — 시작 전 세팅 (온라인 2h)

| 학습 목표 | 내용 |
|-----------|------|

- **실습**: Claude와 첫 대화 — 자기소개 → "내 토론 진행 고민 3가지"를 입력해 응답 비교
- **과제(제출)**: 사전 진단 설문 + "이 과정에서 풀고 싶은 현장 문제 1개"

### M1. Claude 기본기 — 대화·프롬프트·문서화 (4h)

| 학습 목표 | 내용 |
|-----------|------|
| 대화 설계 | 역할 지정·맥락 제공·예시 제시(역할/맥락/형식 3요소) |
| 프롬프트 | 좋은 요청의 구조, 반복 개선(iteration), 출력 형식 지정 |
| 문서화 | Artifacts로 회의록·표·체크리스트 즉시 생성·수정 |
| 프로젝트화 | Projects에 행사 자료를 올려 "맥락을 기억하는" 작업공간 구성 |

- **실습 1**: 같은 요청을 "막연하게" vs "역할·맥락·형식 지정"으로 비교 → 품질 차이 체감
- **실습 2**: 거친 회의 메모 → 깔끔한 회의록(Artifact) 변환
- **산출물**: **나만의 퍼실리테이션 프롬프트 10선** (개인 템플릿)

> 강조점: "마법 주문"을 외우는 게 아니라, **무엇을·왜·어떤 형식으로** 요청하는지를
> 설계하는 능력. 도구가 바뀌어도 남는 역량.

---

### M2. 토론 설계 with AI — 시나리오·질문·자료 (5h)

fkf 숙의 5단계(준비→학습→숙의→합의→환류) 중 **준비·학습** 단계 지원에 집중.

| 학습 목표 | 내용 |
|-----------|------|
| 의제 구조화 | 막연한 주제 → 쟁점·선택지·찬반 논거로 분해 |
| 진행 시나리오 | 시간표·라운드·발문(질문) 세트를 Claude와 공동 설계 |
| 학습자료 | 참가자용 사전 브리핑 자료·용어집 초안 작성 |
| 균형 점검 | 한쪽으로 치우친 발문/자료를 AI로 역검토(편향 체크) |

- **실습**: 제시된 공공의제(예: 지역 생활 SOC 우선순위) → 2시간 숙의 진행 시나리오 완성
- **산출물**: **숙의 진행 시나리오 1부** (의제서 + 발문 세트 + 시간표)
- **연계**: fkf `숙의프로젝트_템플릿_v1.0.0.md`의 Part C(5단계)에 매핑

---

### M3. 현장 진행 보조 — 실시간 요약·논점 추출 (5h)

숙의 5단계 중 **숙의(토론)** 단계의 현장 운용.

| 학습 목표 | 내용 |
|-----------|------|
| 실시간 요약 | 음성/메모 입력 → 라운드별 핵심 요약 즉시 산출 |
| 논점 추출 | 발언에서 쟁점·합의점·이견을 구조화해 화면 공유 |
| 발언 균형 | 발언 분포 메모를 통해 소외 참가자 식별·개입 판단 |
| 중립 진행 | AI 요약을 **퍼실리테이터가 검증 후** 제시하는 워크플로 |

- **실습(롤플레이)**: 4~5인 모의 토론, 1명은 퍼실리테이터+Claude 보조로 진행
- **산출물**: **실시간 진행 키트** (요약 프롬프트·라운드 템플릿·균형 점검표)
- **핵심 원칙**: AI 요약은 **초안**이다 — *읽어주기 전에 사람이 본다.*

---

### M4. 숙의 데이터 분석 — STM 1차 구조화 (5h)

fkf 핵심 기술 **숙의형 데이터 분석(STM, 구조적 토픽 모델링)** 의 입구를 퍼실리테이터가 담당.

| 학습 목표 | 내용 |
|-----------|------|
| 정성 분석 | 토론 기록을 주제·태도·근거 유형으로 코딩 |
| 쟁점 지도 | 의견 군집·대립 축을 시각 구조(표/도식)로 정리 |
| STM 연결 | 정성 코딩 결과를 STM 분석에 넘기기 위한 데이터 정돈 |
| 보고 초안 | 분석 결과 → 의사결정자용 요약 초안 |

- **실습**: M3 토론 기록 → 쟁점 지도 + 1페이지 분석 요약 작성
- **산출물**: **쟁점 지도 + 분석 초안 1부**
- **경계 명시**: 퍼실리테이터는 **1차 구조화**까지. 본격 STM 모델링은 분석 전문가 협업.

---

### M5. 윤리·리스크 — 중립성·개인정보·환각 통제 (3h)

| 학습 목표 | 내용 |
|-----------|------|
| 중립성 | AI가 만든 요약·발문의 편향 점검, 퍼실리테이터의 최종 책임 |
| 개인정보 | 참가자 발언·신상 입력 시 비식별화, 동의 범위 준수 |
| 환각 통제 | 사실 단정 금지, 출처 확인, "모른다"를 유도하는 요청법 |
| 투명성 | 참가자에게 AI 활용 사실을 고지하는 방법·문구 |

- **실습**: 잘못된 활용 사례(개인정보 노출·편향 요약·환각) 진단 및 교정
- **산출물**: **AI 활용 리스크 체크리스트** (현장 부착용 1매)
- **연계**: fkf 보안 원칙 — API 키·크레덴셜·민감정보는 문서·실습에 **절대 미포함**

### M6. 캡스톤 & 인증 — 모의 숙의 + 평가 (8h)

| 단계 | 내용 |
|------|------|
| 사전 준비 | 팀별 의제 선정 → M2~M4 산출물 사전 준비 |
| 모의 숙의 | 30~40분 실제 진행(퍼실 1 + 참가자 + Claude 보조) |
| 산출 | 진행 시나리오·실시간 요약·쟁점 지도·합의 초안 일괄 제출 |
| 평가 | 루브릭 기반 심사(아래 §4) + 동료 피드백 |
| 회고 | 개인별 강점·보완점 정리, 현장 적용 계획서 |

- **산출물**: **캡스톤 패키지**(시나리오+진행기록+분석+보고 초안) + **현장 적용 계획서**

## 7. 과정 차별점 (홍보 메시지 후보)

1. **도구가 아니라 현장** — 프롬프트 암기가 아니라 *숙의 품질(DQI)을 높이는* AI 활용
2. **사람과 AI의 분업** — 사람은 진행·판단, AI는 기록·분석·초안
3. **fkf 솔루션 직결** — 숙의형 데이터 분석(STM)으로 이어지는 1차 구조화 역량
4. **자격으로 끝나지 않음** — 마스터 등급 → 강사·운영진 풀로 성장 경로 제공
