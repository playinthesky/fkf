# 코리아스픽스 직원 일상 업무 가이드

> **버전**: v0.1.0
> **용도**: 피그마/칸바 교재 제작용 원본
> **대상**: 전 직원

---

## 1. 우리가 도달해야 할 목표

### 핵심 비전
> **숙의 컨설팅 노하우를 시스템화하여 누구나 할 수 있게 만든다**

### KPI 5가지

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 주간보고 자동화율 | 80% 초안 자동 생성 | Claude 생성량 vs 수정량 |
| 컨설팅 모델 업데이트 | 주 3회 이상 | GitHub PR 횟수 |
| 데일리 로그 작성률 | 100% | #daily-log 메시지 수 |
| 클라이언트 응답 시간 | 24시간 이내 | 슬랙 응답 기록 |
| 프로젝트 진척 가시성 | 실시간 | 캘린더 업데이트 여부 |

## 2. 매일 할 일 (DAILY)

### 타임라인

| 시간 | 할 일 | 어디서 |
|------|-------|--------|
| 오전 출근 | 데일리 로그 작성 | #daily-log |
| 수시 | 클라이언트 요청 기록 | #client-프로젝트명 |
| 수시 | 일정 변경 시 캘린더 업데이트 | 캘린더 시스템 |
| 수시 | 예산 지출 기록 | #budget |
| 18:00 | AI 학습 결과 확인 (자동) | #ai-learning |

### 데일리 로그 양식

```
[오늘 한 일]
- 

[내일 할 일]
- 

[이슈/도움 필요]
- 
```

### 핵심 포인트
- 출근하면 **가장 먼저** 데일리 로그
- 짧아도 OK, **안 쓰는 건 NG**
- 이슈는 숨기지 말고 바로 공유

---

## 3. 주간 할 일 (WEEKLY)

### 요일별 체크리스트

| 요일 | 할 일 | 어디서 |
|------|-------|--------|
| 월 | 주간 목표 확인 | #weekly-report |
| 화~수 | 프로젝트 진행 | 각 채널 |
| 목 14:00 | 주간보고 초안 수신 (자동) | DM |
| 목~금 | 주간보고 검토/수정 | 구글문서 |
| 금 | HWPX 변환 → 발송 | 이메일 |
| 금 | 다음 주 일정 확정 | 캘린더 |

### 주간보고 플로우

```
목 14:00 Claude가 초안 생성
        ↓
담당 컨설턴트 DM으로 전달
        ↓
검토 및 수정 (목~금)
        ↓
구글문서 → HWPX 변환
        ↓
클라이언트/주관사 발송
```

### 핵심 포인트
- 목요일 오후 **DM 확인 필수**
- 초안 그대로 보내지 않기 → **반드시 검토**
- 금요일 퇴근 전 **다음 주 일정 확정**

## 4. 틈틈이 할 일 (ONGOING)

### 발견 즉시 기록

| 할 일 | 어디서 | 왜 중요한가 |
|-------|--------|------------|
| 컨설팅 노하우 공유 | #ai-learning | AI가 학습함 |
| 클라이언트 피드백 | #client-프로젝트명 | 서비스 개선 |
| 업무 병목/이슈 | #daily-log | 조기 대응 |
| 새로운 사례 발견 | #ai-learning | 모델 강화 |
| 일정 충돌 체크 | 캘린더 | 리소스 최적화 |

### 핵심 포인트
- **"나중에 써야지"는 잊어버림**
- 발견 즉시 슬랙에 메모
- 짧은 한 줄이라도 OK
