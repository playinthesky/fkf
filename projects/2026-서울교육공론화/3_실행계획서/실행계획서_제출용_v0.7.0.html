I 사업 개요

1.1 사업 목적

본 사업은 다음 두 가지 목적을 달성하고자 합니다.

  • 교육정책의 수립·집행 과정에 이해관계자 및 시민이 참여하는 민주적 숙의 과정을 통한 갈등의 사전 예방·해결
  • 정책결정 과정에 다양한 교육 수요자의 참여를 유도하여 의사결정 과정의 투명성과 민주성을 확보하고, 정책 수용성을 제고

1.2 사업 추진 원칙

원칙구현 방식
대표성학생·학부모·교직원·일반시민 등 다양한 교육 수요자가 균형 있게 참여 — 인구통계 층화 추출, 지역 안배, 가족 비중 균형 룰 적용
공정성의제선정·공론 모델 설계·숙의 진행 전 과정의 중립성·객관성 확보 — 외부 위원회 의결, 사전 운영규칙 명시
충분한 숙의온·오프라인 다층 학습 기회 제공 — 자료집 + NotebookLM 온라인 학습 캠페인 + 오리엔테이션 + 전문가 브리핑
투명성전 과정 기록·공개 — 모든 회의 녹취·회의록, 결과의 객관적 분석·공표, 소통광장 상시 게시
실효성도출 결과를 2027년 서울교육정책에 반영 가능하도록 정책권고안·인포그래픽 작성, 교육감 전달 행사 협조

1.3 사업 범위 — RFP 4분야 준수

  1. 의제선정 및 공론 모델 설계 — 공론화의제선정협의체·공론화추진협의체 운영, 절차 설계
  2. 공론과정 운영 — 여론조사, 전문가 의견 수렴, 시민참여단 모집·구성·관리, 숙의 토론회
  3. 홍보 및 사전학습 관리 — 서울교육 소통광장 연계, 참여시민 학습·교육·홍보
  4. 공론화 결과문 및 보고서 작성 — 분석보고서, 정책제안문, 최종보고서, 백서

II 차별화된 접근

본 사업에서 코리아스픽스가 적용하는 4가지 핵심 차별화 솔루션. 모두 우리가 수행한 다른 공론화 사업에서 검증된 방법론이거나, 본 사업에서 새롭게 도입하는 혁신 요소입니다.

CORE METHOD 01
① 숙의형 데이터 분석 (STM, Structural Topic Modeling)
전통적 정성 분석에 구조적 토픽 모델링을 결합한 코리아스픽스 고유 분석 방법론입니다. 시민이 말한 것·쓴 것을 단순 빈도가 아닌 의미 단위의 토픽으로 추출하고, 토픽 간 관계와 변동을 정량·정성 통합으로 분석합니다.
분석 입력
  • 여론조사 주관식 응답 (의제별 1,500명 이상)
  • FGI 4개 그룹 녹취록 (학생·학부모·교사·일반시민)
  • 전문가 인터뷰 녹취록
  • 참여단 숙의 토론장 발언 전체 녹취
  • 소통광장 시민의견·정책설문 응답
분석 산출
  • 핵심 토픽 N개 자동 추출 — 토픽별 키워드·대표 발언
  • 합의·이견 지점 매핑 — 만장 합의 / 다수 / 소수 의견 구분
  • 의견 변동 추적 — 사전 → 학습 후 → 숙의 후 의식 변화
  • 인구통계별 차이 — 연령·성별·계층별 토픽 분포
  • 정성 인용 + 정량 근거 — 결과보고서·백서 본문에 직접 사용
왜 STM인가
  • 빈도 기반 워드클라우드는 의제 본질을 놓침 — 의미 단위 토픽이 핵심
  • 전통 정성 코딩은 분석가 주관에 좌우 — STM은 재현 가능한 객관 근거
  • 1,500명 이상 표본을 사람이 손으로 분류 불가능 — 자동화 필수
적용 실적
  • 충북교육공론화 (2023~2024) — 다년간 STM 분석 적용, 백서 본문에 반영
  • 경기북부특별자치도 설치 공론화 — 시민 의견 STM 분석으로 정책 권고 도출
  • 다수 공공 갈등·숙의 사업에 적용 (코리아스픽스 표준 분석법)
OPERATING MODEL 02
② AX (AI + Human) 협업 운영
의제 분석·자료 가공·기록·분석 전 과정에 AI 도구와 전문 컨설턴트를 결합 운영합니다. AI는 반복·대량 작업을, 사람은 판단·설계·중재를 담당하여 작업 정확성·속도·일관성을 동시에 확보합니다.
AI가 담당하는 일
  • 의제 후보 토픽 자동 추출 (시민 의견·연구 자료 일괄 분석)
  • 전문가 인터뷰 녹취 → 자동 전사 → 핵심 발언 인용 추출
  • 이해관계자 입장별 자료 자동 분류·요약 (자료집 입력)
  • 회의 녹취 → 회의록 초안 자동 생성 (전 회의 적용)
  • 시민의견 자동 분류·감정 분석·STM 분석
  • 참여자 사전·사후 응답 변화 자동 비교
사람만 담당하는 일
  • 의제 설계 — 의제 범위·세부의제 구조화·이해관계 매핑
  • 숙의 절차 설계 — 단계별 진행 방식·세션 디자인
  • 합의문 작성 — 다층 의견의 정책 권고 통합
  • 갈등 중재 — 현장 퍼실리테이션·이해관계자 조정
  • 발주처·위원회 협의·결재 — 모든 결정은 사람이
AX 운영 효과
  • 회의록 작성 시간 70% 단축 (작년 대비)
  • 자료 분석 일관성 향상 — 분석가별 편차 최소화
  • 대량 시민 의견 누락 없이 처리 — 1,500명+ 응답을 정성 수준으로 분석
  • 전 과정 기록 가능 — 발주처 신뢰성 확보
DELIBERATION TECH 03
③ AI 퍼실리테이터 (AI-bdl / 특허 출원 중)
숙의 토론장에서 전문 퍼실리테이터를 실시간 보조하는 코리아스픽스 자체 개발 AI 솔루션입니다. 발언 자동 캡처·논점 추출·팩트체크·발언 균형 모니터링을 동시에 수행하여 퍼실리테이터가 토론 흐름과 갈등 관리에 집중할 수 있도록 합니다.
실시간 기능
  • 발언 캡처·전사 — 모든 발언자의 발언을 실시간 텍스트화
  • 논점 추출 — 발언 흐름에서 핵심 쟁점 자동 정리, 퍼실에 표시
  • 팩트체크 — 발언 내용 중 검증 가능한 사실 주장에 대해 자료집·전문가 인터뷰 근거 매칭
  • 발언 균형 모니터링 — 참가자별 발언량 지니계수 계산, 발언 기회 불균형 시 퍼실에 알림
  • 요약 카드 — 라운드 종료 시 자동 요약 생성, 다음 라운드 입력으로 활용
퍼실리테이터와의 분담
[퍼실리테이터] [AI-bdl] │ │ 토론 진행·중재 ◀─── 논점·균형 알림 ◀── 발언 캡처·분석 참여자 격려 │ 감정·갈등 관리 │ 합의 유도 ───▶ 요약 요청 ───▶ 라운드 요약·인용 추출 │ │ ▼ ▼ 토론 흐름 통제 실시간 기록·분석
왜 별도 기록자 운영을 하지 않는가
  • 기록자가 퍼실 옆에서 단순 받아쓰기를 하면 토론 흐름이 끊김 (작년 운영 결과 확인)
  • AI-bdl이 발언 캡처를 전담 → 사람 기록자는 갈등 관리·심층 인터뷰에 재배치
  • 녹취 + AI 전사 + 사후 사람 검수 3단계로 정확성·맥락 모두 확보
기대 효과 (DQI 영향)
  • 발언 균형 지니계수 ≤ 0.3 달성률 향상
  • 근거 활용(팩트 인용률) ≥ 60% 달성
  • 토론 시간 활용도 향상 — 같은 시간에 더 깊은 토론
CIVIC LEARNING 04
④ NotebookLM Meetup — 시민 학습 보강 캠페인
자료집만으로는 부족한 의제 학습을 보강하기 위한 Google NotebookLM 기반 온라인 학습 캠페인입니다. 본 사업에서 처음 도입하는 혁신 요소로, 발주처·구글코리아와 공동 기획하여 시민 참여단의 의제 이해도를 한 단계 끌어올립니다.
NotebookLM이 무엇인가

Google이 개발한 AI 기반 학습·연구 도구. 사용자가 업로드한 자료(PDF·문서)를 기반으로만 답변하며, 자료 외 추론은 하지 않아 환각(hallucination) 위험을 차단합니다. 시민이 자료집을 능동적으로 학습하고 자신의 페이스로 질문할 수 있는 도구로 적합합니다.

운영 흐름
┌─────────────────────────────────────────────────────┐ │ STEP 1. 사전 준비 │ │ • 의제 확정 직후 — 자료집·전문가 인터뷰·연구보고서를 │ │ NotebookLM Notebook에 업로드 │ │ • 발주처·구글코리아와 공동 기획 │ └─────────────────────────────────────────────────────┘ ▼ ┌─────────────────────────────────────────────────────┐ │ STEP 2. 시민 참여단 학습 (OT 직후 ~ 본 토론 전) │ │ • Notebook 접속 링크 발송 │ │ • 시민이 자기 속도로 자료 학습 │ │ • 궁금한 점을 NotebookLM에 질문 │ │ • 자료에 근거한 답변 즉시 수신 │ └─────────────────────────────────────────────────────┘ ▼ ┌─────────────────────────────────────────────────────┐ │ STEP 3. 본 토론 입력 │ │ • 시민 질문 로그 → 자주 묻는 질문 분석 │ │ • 전문가 패널에서 우선 다룰 쟁점으로 활용 │ │ • 토론 시작 전 모든 시민이 동일한 학습 수준 도달 │ └─────────────────────────────────────────────────────┘
혁신성
  • 전통 공론화의 사전 학습은 오프라인 OT 한 번 — 시민 부담·이해도 한계
  • NotebookLM은 24시간 시민 페이스 학습 — 직장인·학생·학부모 모두 접근 가능
  • 질문 로그가 본 토론 입력 — 시민이 진짜 궁금한 것을 다룸
  • 구글코리아 협조로 안정 인프라 — 별도 서버 부담 없음
발주처 협의 사항
  • 도입 동의 및 공동 기획 회의
  • 구글코리아 협조 요청 공문 발송 (코리아스픽스가 초안 작성)
  • "서울교육공론 NotebookLM Meetup" 브랜드 명칭 협의
  • 운영 후 학습 로그 데이터 활용·보관 정책 협의

공통 원칙: 위 4가지는 코리아스픽스의 모든 숙의 컨설팅에 내장된 표준 솔루션입니다. 본 사업에서도 동일하게 적용되며, ④ NotebookLM Meetup만 본 사업에서 새롭게 도입하는 혁신 요소입니다.

그룹대상인원시간주제

다. 공론화의제선정협의체 운영 (1~2회, 10명)

회차안건산출물
1차 (1회 운영 시 단독 / 2회 운영 시 사전)의제 후보 검토, 시민·전문가 의견 보고, 후보 압축 토론의제 후보 보고서, 1차 압축 명단
2차 (선택)최종 의제 선정 토론·투표최종 의제 확정문
의사결정 모델
  1. 1단계 — 만장일치 추구: 위원 토론을 통한 합의 시도
  2. 2단계 — 다수결: 합의 불성립 시 위원 2/3 이상 찬성
  3. 3단계 — 위원장 결정: 다수결도 부결 시 위원장이 최종 의제 결정 (위임)
위원 인선 협력
  • 학계·시민·교사·학부모·전문가 균형 (각 영역 2명 내외)
  • 의제 분야별 전문성 확보 (정책·교육·갈등관리)
  • 발주처와 협의하여 위원 추천·위촉

III-2. 공론과정 운영

  • 영상 — 단계별 숙의공론 과정 영상 (편집본 + 원본)
