## 3. 단계별 활동 (시점 표시 ❌, 활동 정의만)

### 3.1 사전 준비 (킥오프 직후)
- 사내 착수 회의
- 카운터파트너 인사 (재은 명함 / 본인 인사)
- **보험 가입·증서 발급** → 제출 (작년 누락)
- **AI-bdl(AI 화백) 환경 셋업 시작**
- **NotebookLM 환경 셋업 + 구글코리아 협조 요청** 발송
- **소통광장 아이디 신청** (작년 미활용)
- **회의록 SOP 가동** (녹취 + 작성 앱)
- **SAFETY CODE 1차 점검**

### 3.2 의제 의견수렴
- 시민 의제 의견수렴 채널 운영 (소통광장 + 자체 폼)
- **PR 단계 1 (서울교육공론화 소개)** 동시 가동
- 의견 모니터링 + 중간 분석
- 부서 의견 조회 (교육청 측)

### 3.3 의제 선정
- 부서 의견 종합
- **의제선정협의회 위원 위촉** (클라이언트와 협의해 인선)
- 의제선정협의회 회의 + 투표
- **최종 의제 확정**

### 3.4 공론화추진협의회 가동
- 추진협의회 위원 위촉 (클라이언트 협의)
- **추진협 1차 회의**: 의제명 확정, 운영 설계, 일정 공유, **AI-bdl 사전 안내** (작년 기록자 분리 요구 사전 차단)
- 전문가 인터뷰 (의제 관련)
- **추진협 2차 회의**: 설문 문항·자료집·세부의제
- 사전조사 + 참여단 모집 (**가족 비중 필터 적용**)
- 참여단 층화 추출
- **추진협 3차 회의**: 숙의 토론 운영 설계
- 자료집 제작 + 검토
- 퍼실 모집 (학부모 / 전문 별도) + 교육
- **NotebookLM Meetup 시범 → 본 가동**
- **PR 단계 2 (의제 선정 안내 + 의견수렴 2차) → PR 단계 3 (숙의 과정 공유)**

### 3.5 숙의 본 회차 (1일 종일, 토요일)
- 사전 일정 (목/금): 공론장 조성 + 기술 리허설 + 드레스 리허설
- 당일 (토): **숙의토론 진행** (AI-bdl 가동, **SAFETY CODE 운영** — 안전요원·응급·주차)
- 테이블 설계: 컨퍼런스룸 인원 ↓ → 테이블 수 ↑

### 3.6 환류 (숙의 직후 ~ 12/31)
- 결과 즉시 분석
- **참가자 1:1 결과 보고 메시지 + 땡큐 메시지 발송**
- **교육감 명의 메시지** (글로벌팀 제안 채택 검토)
- **PR 단계 4 (결과 발표)** 가동
- 백서 초안 제작 (글로벌팀, 가독성 우선)
- DQI 측정 + 평가보고서
- **추진협 4차 회의** (백서 종합 보고)
- **결과보고회 참가 + 촬영 + 홍보** (작년 불참)
- 백서 최종 + 모든 산출물 납품
- 종료
